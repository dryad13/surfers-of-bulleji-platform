# MVP Scope

## MVP definition

The first version is a **private functional prototype** built to pitch the community.

It should feel real, but it must not pretend to be official before approval.

## MVP goals

1. Show the community what a real platform could look like.
2. Make the story understandable within 10 seconds.
3. Create trust through public press/social proof.
4. Demonstrate support and equipment inquiry flows.
5. Demonstrate surf session request flow.
6. Demonstrate the Boardroom / board tracker concept.
7. Avoid legal, consent, donation, and safety risks before approval.

## Non-goals for MVP

Do not build these yet:

```txt
Real payment collection
Full admin dashboard
User accounts
AI chatbot
Calendar booking automation
WhatsApp automation
Public minor profiles
Live donation tracker
Complex CMS
```

These can come after community approval.

## MVP pages

### 1. Home

Purpose: explain the movement immediately.

Sections:

- Prototype banner
- Cinematic hero
- Short origin story
- Impact stats
- Support CTA
- Surf session CTA
- Boardroom preview
- Press/social proof
- Gallery preview
- Footer

Hero copy direction:

> Pakistan's first surf community, born on the shores of Karachi.

Subcopy direction:

> Self-taught surfers from Bulleji are building Pakistan's surf future with repaired boards, shared gear, and relentless love for the sea.

### 2. Story

Purpose: show the human journey without poverty marketing.

Sections:

- Born from the sea
- Self-taught, self-made
- Broken boards, unbroken spirit
- The dream ahead
- Call to support

### 3. Meet the Surfers

Purpose: humanize the community.

MVP rule:

Use only public information and placeholders until approval.

Card fields:

```txt
Name
Role
Short bio
Quote
Image placeholder
Approval status
```

### 4. Support

Purpose: convert emotional attention into practical help.

Support categories:

```txt
Sponsor a board
Repair a board
Donate safety gear
Donate surf gear
Sponsor a young surfer
Partner with the community
Volunteer skills/time
```

MVP behavior:

- Inquiry form only
- No live payment collection
- Donation language should say “support inquiry”

### 5. Boardroom / Board Tracker

Purpose: make support tangible.

Board card fields:

```txt
Name
Type
Status
Condition
Needs
Sponsor status
Estimated cost placeholder
Image placeholder
```

Sample statuses:

```txt
In use
Needs repair
Needed
Retired
Sponsored
```

### 6. Request a Surf Session

Purpose: show revenue potential without overpromising.

MVP behavior:

- Manual inquiry only
- No payment
- No calendar guarantee
- Confirmation happens later on WhatsApp

Required form fields:

```txt
Name
WhatsApp number
Preferred date
Number of people
Can you swim?
Experience level
Message
Emergency contact optional for demo
Consent checkbox
```

### 7. Media

Purpose: build legitimacy.

Sections:

- Press links
- Documentary references
- Media inquiry form
- Press kit placeholder
- Social links

### 8. Contact

Purpose: route interested people.

CTAs:

```txt
WhatsApp pending official number
Instagram
Facebook
Support inquiry
Session inquiry
Equipment donation
Media inquiry
```

## Functional MVP checklist

- [ ] Responsive homepage
- [ ] Prototype banner
- [ ] Story page
- [ ] Boardroom tracker using static data
- [ ] Support inquiry form
- [ ] Equipment donation form
- [ ] Surf session request form
- [ ] Press/media page
- [ ] Contact page
- [ ] Basic SEO metadata
- [ ] Mobile-first layout
- [ ] Compressed images
- [ ] Safety/consent disclaimers

## Demo content rule

Every unapproved piece of content must be either:

1. Clearly public information
2. Placeholder content
3. Marked as pending approval

## Suggested build sequence

```txt
1. Scaffold Astro + Tailwind
2. Add global layout and theme
3. Build homepage
4. Add static data files
5. Build Boardroom page
6. Build Story page
7. Build Support page + form
8. Build Booking page + form
9. Build Media and Contact pages
10. Polish mobile, copy, and pitch flow
```
